1. Kiedy Git wykona fast-forward, a kiedy powstaje merge commit? 

Fast-forward: kiedy gałąź główna nie ma nowych commitów od momentu utworzenia gałęzi pobocznej

Merge commit: kiedy gałęzie się rozeszły  i git musi commitem je połączyć

2. Czym w praktyce różni się merge od rebase? 

Merge: opisuje dokłądnie historie pracy i pokazuje kiedy gałęzie się rozeszły i kiedy zostały połączone.

Rebase: pokazuje historię pracy w sposób liniowy, przez co historia jest bardziej czytelna.
